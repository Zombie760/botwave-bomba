# Contributing to BotwaveBomba

## Code of Conduct

This project follows a simple rule: **Be Undeniable**. That means evidence over claims, receipts over conjecture, and respect for the operator's constraints.

## How to Contribute

### Reporting Bugs

Open an issue with:
- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Environment details (browser, OS)

### Proposing Features

Before opening a feature request:
1. Check existing issues and README
2. Ensure the feature aligns with the project mission
3. Provide use cases and evidence

### Pull Requests

1. Fork the repo and create a feature branch
2. Follow existing code style (TypeScript, Bun runtime)
3. Run `bun run validate` before submitting
4. Update docs if behavior changes
5. Keep commits atomic and descriptive

## Development Setup

```bash
git clone https://github.com/zombie760/botwavebomba.git
cd botwavebomba
bun install
bun run dev
```

## Commit Standards

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `chore:` maintenance
- `refactor:` code change
- `test:` test addition
- `ci:` CI/CD changes

## Questions?

Open an issue or reach out via the site contact.
